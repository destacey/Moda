# Logging In

Moda supports two interactive authentication methods. Your administrator will tell you which one to use.

## Microsoft Entra ID (Azure AD)

If your organization uses Microsoft Entra ID:

1. Navigate to your Moda instance URL
2. Click **Sign in with Microsoft**
3. Enter your organizational credentials
4. You'll be redirected to Moda after successful authentication

## Local Authentication

If your instance uses local authentication:

1. Navigate to your Moda instance URL
2. Select **Email & Password**
3. Enter your **username** and **password**
4. Click **Sign In**

If this is your first login, you may be prompted to change your password before continuing.

## API Access with Personal Access Tokens

For scripts, automations, and integrations that need to access the Moda API without interactive login, you can use a **Personal Access Token (PAT)**.

PATs are created from your [Profile](./your-profile.mdx#personal-access-tokens) page and included in API request headers. See [Your Profile > Personal Access Tokens](./your-profile.mdx#personal-access-tokens) for details on creating tokens, and the [API Reference](../reference/api.mdx#personal-access-tokens-pats) for how to use them.

## Troubleshooting

- **Account locked?** — Contact your administrator to unlock your account. Accounts are locked after multiple failed login attempts.
- **Forgot password?** — Contact your administrator for a password reset (local authentication only). You'll be required to set a new password on next login.
- **Can't access?** — Verify you have an active account with your administrator.
- **Must change password?** — Local users created by an administrator are required to set a new password on first login. This is a one-time prompt.
