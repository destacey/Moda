# MCP Server

Moda provides an [MCP (Model Context Protocol)](https://modelcontextprotocol.io) server that lets AI assistants like Claude interact directly with your Moda instance. The MCP server exposes Moda's API as structured tools that AI agents can call to read and manage your delivery data.

**Package:** [`@modanpm/moda-mcp`](https://www.npmjs.com/package/@modanpm/moda-mcp)

## What Can It Do?

The MCP server exposes 50+ tools across Moda's core modules:

| Module | Capabilities |
|--------|-------------|
| **[PPM](../user-guide/ppm/index.mdx)** | List/view portfolios, programs, projects, phases, tasks. Create and update tasks. View project plan tree and summary metrics. Get critical path. Manage task dependencies. |
| **[Planning](../user-guide/planning/index.mdx)** | List/view planning intervals, iterations, objectives, risks. Get PI predictability and health reports. View calendars. |
| **[Roadmaps](../user-guide/planning/roadmaps.mdx)** | List/view roadmaps and their items (activities, milestones, timeboxes). |
| **[Organizations](../user-guide/organizations/index.mdx)** | List/view teams and users. |

## Prerequisites

1. A running Moda instance with API access
2. A **[Personal Access Token (PAT)](./your-profile.mdx#personal-access-tokens)** — create one from your profile page
3. **Node.js 20+** installed on your machine

## Setup

### Claude Desktop

Add the following to your Claude Desktop configuration file:

- **macOS:** `~/Library/Application Support/Claude/claude_desktop_config.json`
- **Windows:** `%APPDATA%\Claude\claude_desktop_config.json`

```json
{
  "mcpServers": {
    "moda": {
      "command": "npx",
      "args": ["-y", "@modanpm/moda-mcp"],
      "env": {
        "MODA_API_BASE_URL": "https://your-moda-instance.com",
        "MODA_API_KEY": "your-personal-access-token"
      }
    }
  }
}
```

### Claude Code

Add a `.mcp.json` file to your project root:

```json
{
  "mcpServers": {
    "moda": {
      "command": "npx",
      "args": ["-y", "@modanpm/moda-mcp"],
      "env": {
        "MODA_API_BASE_URL": "https://your-moda-instance.com",
        "MODA_API_KEY": "your-personal-access-token"
      }
    }
  }
}
```

Or set environment variables in your shell and omit the `env` block:

```bash
export MODA_API_BASE_URL="https://your-moda-instance.com"
export MODA_API_KEY="your-personal-access-token"
```

### VS Code / Cursor

Add to your `.vscode/mcp.json` or workspace settings:

```json
{
  "inputs": [
    {
      "id": "modaBaseUrl",
      "description": "Moda API base URL",
      "type": "promptString"
    },
    {
      "id": "modaApiKey",
      "description": "Moda API key (Personal Access Token)",
      "type": "promptString",
      "password": true
    }
  ],
  "servers": {
    "moda": {
      "type": "stdio",
      "command": "npx",
      "args": [
        "-y", "@modanpm/moda-mcp",
        "--base-url", "${input:modaBaseUrl}",
        "--api-key", "${input:modaApiKey}"
      ]
    }
  }
}
```

## Configuration

The MCP server accepts configuration via environment variables or CLI arguments:

| Environment Variable | CLI Argument | Description |
|---------------------|-------------|-------------|
| `MODA_API_BASE_URL` | `--base-url` | The base URL of your Moda API instance |
| `MODA_API_KEY` | `--api-key` | Your [Personal Access Token](./your-profile.mdx#personal-access-tokens) |

CLI arguments take precedence over environment variables.

## Agent Skills

For Claude Code users, three pre-built skills provide guided workflows:

- **`/moda-ppm`** — Portfolio, program, and project operations
- **`/moda-pi`** — Planning intervals, iterations, objectives, and risks
- **`/moda-roadmaps`** — Roadmap exploration with activities and milestones

Install the skills:

```bash
npx skills add destacey/moda
```

## Security

The MCP server authenticates using your [Personal Access Token](./your-profile.mdx#personal-access-tokens). The token inherits your permissions — the MCP server can only access what you can access.

- Store your API key securely — don't commit it to version control
- Use environment variables rather than hardcoding in config files
- Revoke tokens from your [profile page](./your-profile.mdx#personal-access-tokens) if compromised
