# Your Profile

After logging in, you can manage your profile and account settings.

## Viewing Your Profile

Navigate to your profile by clicking your avatar or name in the top-right corner of the application.

## Profile Settings

- **Display Name** — Your name as shown throughout the application
- **Email** — Your email address
- **Login Provider** — How you authenticate (Microsoft Entra ID or Local). This is set when your account is created and cannot be changed.

## Changing Your Password

> Only available for users with the **Moda (Local)** login provider. Microsoft Entra ID users manage their passwords through their organization's Azure AD.

1. Navigate to your profile
2. Click **Change Password**
3. Enter your current password
4. Enter and confirm your new password
5. Click **Save**

If this is your first login and you see a **Must Change Password** prompt, you'll need to set a new password before continuing.

## Personal Access Tokens

**Personal Access Tokens (PATs)** allow you to authenticate with the Moda [API](../reference/api#personal-access-tokens-pats) without using your username and password. They are useful for scripts, automations, and integrations.

### Creating a Token

1. Navigate to your profile
2. Click **Personal Access Tokens**
3. Click **Create Token**
4. Enter a **Name** for the token (to help you remember what it's used for)
5. Optionally set an **Expiration Date**
6. Click **Create**
7. **Copy the token immediately** — it will not be shown again

### Using a Token

Include the token in the `Authorization` header of your API requests:
```
Authorization: Bearer {your-personal-access-token}
```

See the [API Reference](../reference/api#personal-access-tokens-pats) for full details on API authentication.

### Managing Tokens

- View all your active tokens on the Personal Access Tokens page
- **Revoke** a token when it's no longer needed — this immediately invalidates it
- Expired tokens are automatically invalid and can be cleaned up

PATs inherit your permissions — they can do anything you can do in the API. Treat them like passwords and don't share them.
