# Work Management

The Work Management domain is the core of Moda, providing tools to create, track, and manage work across your organization.

## Key Concepts

### Work Items

Work items are the fundamental unit of work in Moda. They represent tasks, bugs, user stories, features, or any other type of work that needs to be tracked.

Each work item has:
- **Title** - A brief description of the work
- **Type** - The category of work (defined by the work process)
- **Status** - Current state in the workflow
- **Priority** - Relative importance
- **Assignee** - The person responsible
- **Parent** - Optional hierarchical relationship to another work item

### Work Processes

A work process defines the types of work items and their workflow states available in a workspace. Work processes provide:
- **Work Types** - Categories of work items (e.g., Epic, User Story, Task, Bug)
- **Workflows** - State machines defining valid status transitions
- **Type Hierarchy** - Parent-child relationships between work types

### Workspaces

Workspaces are containers for work items that use a specific work process. They provide scope and organization for related work.
