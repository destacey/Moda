# Work Processes

Work processes define the structure and workflow for work items within a workspace.

## What is a Work Process?

A work process is a template that defines:
- **Work Types** - The categories of work items available (e.g., Epic, Feature, User Story, Task, Bug)
- **Workflows** - The state machines that define valid status transitions for each work type
- **Type Hierarchy** - Which work types can be parents of which other work types

## Workflows

Each work type has an associated workflow that defines:
- **States** - The possible statuses (e.g., New, Active, Resolved, Closed)
- **Transitions** - Valid moves between states
- **State Categories** - Groupings (Proposed, Active, Done, Removed) for reporting

## Managing Work Processes

Work processes are configured in **Settings > Work Management > Work Processes**.

> **Note:** Changes to a work process affect all workspaces using that process.
