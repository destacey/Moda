# Managing Work Items

## Viewing Work Items

Work items can be viewed in several ways:
- **List View** - Tabular view with sorting and filtering
- **Detail View** - Full work item details with activity history

## Editing Work Items

Click on a work item to open its detail view. Editable fields include title, description, priority, assignee, and custom fields defined by the work process.

> **Note:** Work items with **Managed** ownership (synced from external systems like Azure DevOps) are read-only in Moda.

## Transitioning Status

Work items follow the workflow defined by their work process. Available status transitions depend on the current state. To change status:

1. Open the work item
2. Select the new status from the available transitions
3. The change is saved automatically

## Linking Work Items

Work items can be linked to other entities in Moda:
- **Parent/Child** - Hierarchical relationships between work items
- **Links** - Cross-entity associations (to objectives, teams, etc.)
