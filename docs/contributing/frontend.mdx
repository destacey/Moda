# Frontend Development

The Wayd frontend is built with **Next.js 16** (App Router), **React 19**, **TypeScript**, and **Ant Design**.

## Project Structure

```
wayd.web.reactclient/
  src/
    app/              # Next.js App Router pages and layouts
    components/       # Shared components
      common/         # Generic reusable components
      contexts/       # React context providers (auth, theme, messaging)
      hoc/            # Higher-order components (feature flags)
    hooks/            # Custom React hooks
    services/         # API clients and services
      wayd-api.ts     # NSwag-generated TypeScript client
      clients.ts      # Client factory functions
    store/            # Redux Toolkit store and slices
    styles/           # Global styles and CSS modules
```

## API Calls

**Always use the NSwag-generated typed client:**

```typescript
import { getWorkItemsClient } from '@/src/services/clients'

const client = await getWorkItemsClient()
const workItems = await client.getList(workspaceId)
```

**Never use `authenticatedFetch()` directly** unless the generated client doesn't have the method yet.

## Styling

Use **Ant Design theme tokens**. Never hardcode colors.

Prefer CSS variables in CSS modules:
```css
/* good */
.header { color: var(--ant-color-primary); }

/* bad */
.header { color: #1677ff; }
```

Only use `theme.useToken()` when you need token values in JavaScript logic.

## State Management

- **Redux Toolkit** for global application state
- **React Context** for auth, theme, and messaging
- **Local state** (`useState`) for component-specific state

## PWA (Progressive Web App)

The client app is installable as a PWA, providing a native app-like experience with app shell caching and network-first API strategy.

**Key files:**

| File | Purpose |
|------|---------|
| `src/app/manifest.ts` | Web app manifest (name, icons, screenshots, theme color) |
| `src/app/sw.ts` | Service worker source (precaching + runtime caching) |
| `src/app/serwist/[path]/route.ts` | Turbopack route handler that builds and serves the SW |
| `src/config/theme/theme-constants.ts` | Shared brand color used by manifest and light theme |
| `public/icons/` | PWA icons (192x192, 512x512) |
| `public/screenshots/` | Install dialog screenshots (desktop and mobile) |

**Tooling:** [Serwist](https://serwist.pages.dev) (`@serwist/turbopack`) — handles service worker compilation and precache manifest injection via a Next.js route handler.

**Development:** The service worker is only active in production builds. To test PWA features locally, run `npm run build && npm start`.

**Caching strategy:** `defaultCache` from Serwist provides network-first for navigations and API calls, and cache-first for static assets (JS, CSS, images, fonts).
