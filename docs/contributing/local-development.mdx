# Local Development

## Running with Aspire (Recommended)

.NET Aspire provides a unified dashboard for all services:

```bash
cd Wayd.AppHost
dotnet run
```

**Benefits:**
- Unified telemetry dashboard (traces, metrics, logs)
- Automatic OpenTelemetry instrumentation
- Dynamic port assignment with service discovery

## Common Tasks

### Adding a Database Migration

```bash
dotnet ef migrations add <MigrationName> \
  --project "Wayd.Infrastructure/src/Wayd.Infrastructure.Migrators.MSSQL" \
  --startup-project "Wayd.Web/src/Wayd.Web.Api"
```

### Running Tests

```bash
# All tests
dotnet test Wayd.slnx

# Specific project
dotnet test "Wayd.Services/Wayd.Work/tests/Wayd.Work.Application.Tests/Wayd.Work.Application.Tests.csproj"

# Filtered tests
dotnet test --filter "FullyQualifiedName~ProjectServiceTests"

# Frontend tests
cd Wayd.Web/src/wayd.web.reactclient
npm test
```

### Architecture Tests

```bash
dotnet test Wayd.ArchitectureTests/Wayd.ArchitectureTests.csproj
```

These tests enforce Clean Architecture dependency rules across the solution.

## VS Code Launch Configurations

- **Compound: Launch Wayd on Host** - Runs API and client natively
- **Compound: Launch Wayd with Compose** - Full stack via Docker Compose

## Debugging

- Backend: Attach to the `Wayd.Web.Api` process or use VS Code launch config
- Frontend: Use browser DevTools or VS Code debugger
- Logs: Check Aspire dashboard or Seq (Docker) for structured logs
